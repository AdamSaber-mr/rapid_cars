We are continuing an existing RapidCars website. The homepage is already finished and approved, and it should not be redesigned. Please keep the same overall brand feeling and design language as the homepage. You have creative freedom to choose styling details (spacing, card layout, typography, small UI details), as long as it stays consistent with the existing homepage look and maintains the same “80% minimal / 20% energy” vibe. Please avoid typical AI clichés (random emojis, overly playful decorations, unnecessary gradients, fake badges, or clutter). Keep it modern, clean, and professional for a young audience.

Now create two new pages: Aanbod and Contact, and implement a clear customer flow from “view cars” to “send a request”.

The website’s goal is simple: visitors come to see which cars RapidCars has and then quickly request availability. The target audience is mostly young people, so the process must be fast and low-friction. Many users prefer messaging via WhatsApp or Snapchat rather than filling out long forms. Therefore, the primary conversion flow should prioritize messaging first, with a short form as a backup option.

1) Aanbod Page (Car Listing)

Create an “Aanbod” page that matches the style of the homepage. At the top, include a clean heading and short subtitle (minimal text, because users scroll fast). The main content must be a grid or carousel-like list of car cards. Use real-looking placeholder data and images (sports cars similar to Audi RS3, Golf R, BMW M2/M3, etc.). Each car card should contain an image, car name, and a short one-line spec summary (e.g., horsepower, drivetrain, or 0-100). Keep the layout clean and readable, consistent with the homepage.

Each car card must include a clear button like “Vraag beschikbaarheid”. When the user clicks this button, do NOT immediately send them to a long form page. Instead, open a modern modal/panel/contact sheet that keeps the user in the flow.

2) “Vraag beschikbaarheid” Modal/Panel (Main Conversion Flow)

When the user clicks “Vraag beschikbaarheid”, show a panel/modal that clearly mentions the selected car name at the top (for example: “Vraag beschikbaarheid — Audi RS3”). Under that, offer the user three contact options in this order:

WhatsApp (primary, most prominent button)

Snapchat (secondary option)

Formulier (small link/button like “Liever mailen? Vul het formulier in” that expands a short form inside the modal or takes them to the contact page with the car preselected)

For WhatsApp, design the button so it opens WhatsApp with a pre-filled message that includes the car name. Example message: “Hi RapidCars, ik wil graag beschikbaarheid checken voor de [AUTO]. Voor periode: [DATUM/PERIODE]. Kunnen jullie mij helpen?” (If dates are not selected yet, keep the period part optional or blank.)

For Snapchat, show a button and also show the Snapchat username with a “Copy” button (because deep linking to Snapchat can be inconsistent). For example: show “Snapchat: RapidCarsNL” with a copy-to-clipboard action.

The panel/modal must feel modern, fast, and professional, with minimal steps and clear hierarchy. Keep the design consistent with the homepage but you may decide the best layout, animations, and spacing.

3) Contact Page (Secondary Conversion + Information)

Create a full “Contact” page accessible from the navbar. This page should include:

A short intro: users can contact RapidCars for availability or questions

Quick contact blocks/buttons for WhatsApp and Snapchat (same as in the modal)

A short contact form (not long). Keep it minimal, ideal for young users.

Form fields should be short and practical, for example:

Name

Phone number (important for young users)

Email (optional if you want, but phone is key)

Car (dropdown or prefilled if coming from a specific car)

Rental period/date

Optional message

Include a clear submit button like “Verstuur aanvraag”. The form submission should conceptually send to RapidCars email (no backend needed in prototype, just design the flow).

4) Flow Requirements

From homepage, users can click to “Aanbod”.

On “Aanbod”, clicking “Vraag beschikbaarheid” opens the modal/panel with WhatsApp/Snapchat/Form options.

If the user chooses the form route, the selected car should be pre-filled automatically (or at least shown clearly).

The entire experience should feel fast, modern, and low-effort.

5) Design Freedom (Important)

You have freedom to choose the best UI style for these pages as long as it matches the existing homepage. Keep the same overall quality level. Use the existing homepage as reference for typography and vibe. Make the pages feel like a natural extension of the homepage design.

Deliver a clickable prototype for the Aanbod page, Contact page, and the modal/panel interaction from the Aanbod cards.
